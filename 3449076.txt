En este curso de Git y GitHub, aprendí muchos conceptos importantes para el control de versiones y la colaboración en proyectos. Primero, entendí la importancia de Git como sistema de control de versiones distribuido, que me permite llevar un registro detallado de los cambios en mis proyectos. Aprendí a configurar Git en mi máquina, crear repositorios locales y remotos, y realizar commits para guardar los cambios. Además, ahora soy capaz de utilizar ramas para trabajar en diferentes características de un proyecto sin afectar la versión principal.

En GitHub, aprendí a subir mis repositorios locales, colaborar con otros usuarios mediante pull requests y gestionar proyectos compartidos. También entendí la importancia de los conflictos de merge y cómo resolverlos. El curso incluyó ejercicios prácticos que me ayudaron a consolidar conceptos como el uso de comandos como: git add, git commit, git push, git pull, git merge y la creación de forks y branches. Ahora me siento mucho más cómodo trabajando en proyectos en equipo y gestionando mis propios proyectos con Git y GitHub.
